Customization seams

Pi is extension-shaped on purpose

Extensions, SDK options, resources, RPC, custom providers, custom tools, custom messages, prompt templates, skills, themes, and TUI components are all first-class integration points.

Extension loading

Extensions are TypeScript/JavaScript factory modules loaded from direct files, extension directories with index.ts/js, or packages declaring pi.extensions. The loader aliases Pi package imports so extensions can import public APIs consistently in source, package, or binary mode.

Load errors are diagnostics, not fatal crashes. Tool and flag conflicts are reported; first registration generally wins in runtime lookup.

Extension API registration surface

Events

pi.on(event, handler) hooks resource discovery, session lifecycle, context, provider request/response, agent lifecycle, messages, tools, input, model/thinking selection, compaction, tree operations, and session shutdown.

Tools

registerTool() adds LLM-callable tools with schema, execution, prompt snippets/guidelines, argument preparation, sequential mode, and TUI renderers.

Commands

registerCommand() adds slash commands with descriptions, optional argument completions, and handlers.

Shortcuts and flags

registerShortcut() adds key shortcuts. registerFlag() declares CLI flags that are validated after extension load.

Renderers and messages

registerMessageRenderer() displays custom message types. sendMessage() injects custom messages into the session/LLM context.

Providers

registerProvider() can add OAuth, headers, models, base URLs, auth behavior, or a custom streamSimple implementation.

Extension event classes

Resource/session

resources_discover, session_start, session_before_switch, session_before_fork, session_before_compact, session_compact, session_shutdown, tree events.

Agent/model

context, before_provider_request, after_provider_response, before_agent_start, agent_start/end, turn_start/end, message_start/update/end, model/thinking selection.

Input/tools

input can transform/handle user text. tool_call can block execution. tool_result can patch result content/details/error/terminate.

UI extension surface

Interactive mode supplies a rich ExtensionUIContext: select/confirm/input dialogs, notifications, terminal input listeners, status and working indicators, widgets, custom footer/header/title, editor control, autocomplete wrapping, theme APIs, overlays, and temporary focused components. RPC supplies protocol-backed versions for dialogs and simple status/widgets; print mode cannot host interactive TUI UI.

Resource extension points

  • Packages: bundle extensions, skills, prompts, and themes in npm/git/local packages.
  • Skills: add model-invocable markdown capabilities and optional slash commands.
  • Prompt templates: add reusable slash-command prompt expansions.
  • Themes: add JSON color systems.
  • Context/system prompt files: alter global/project prompt context without code.

SDK extension points

createAgentSession() accepts custom cwd/agentDir, auth/model/settings/resource/session managers, explicit model/thinking/scoped models, noTools, tool allowlists, custom tools, injected resource loaders, and session start events. DefaultResourceLoaderOptions also accepts inline extension factories, extra paths, disable flags, prompt overrides, and override callbacks for loaded resources.

Low-level extension points

Agent

Inject streamFn, convertToLlm, transformContext, getApiKey, beforeToolCall, afterToolCall, queues, transport, retry caps, thinking budgets.

pi-ai

Register or unregister API providers, image providers, OAuth providers, and custom model catalogs. Dispatch remains keyed by model.api.

TUI

Build custom components that implement render(width) and optional handleInput(); use overlays, focus, keybindings, and cursor markers.
