How Pi fits together

Pi is a layered agent harness: provider-neutral LLM streaming in packages/ai, a generic evented agent loop in packages/agent, a batteries-included coding-agent runtime in packages/coding-agent, and UI surfaces built on packages/tui, RPC, print/JSON, SDK, and web components.

Most important seam: AgentSession is the product boundary. It wraps the low-level Agent with sessions, resources, extensions, built-in tools, compaction, retry, model/auth management, and UI-facing events.

Package responsibilities

packages/ai

Unified LLM API. Defines model/provider types, message/tool schema, streaming event protocol, provider registry, generated model catalog, auth environment detection, and built-in provider registration.

packages/agent

Provider-agnostic agent core. Owns transcript state, the agent loop, steering/follow-up queues, tool execution, lifecycle events, and the exact LLM-call boundary.

packages/coding-agent

Pi product runtime. Loads config/resources, creates sessions, manages auth/model selection, builds prompts, registers tools/extensions, persists JSONL sessions, and exposes interactive/print/RPC/SDK modes.

packages/tui

Terminal UI framework. Provides raw terminal input, focus, components, overlays, editor, differential rendering, image support, and keybinding primitives.

packages/web-ui

Browser component library built around pi-agent-core and pi-ai. It is a web UI toolkit, not a Telegram/Discord adapter.
