Resource loading

Configuration is layered by scope, then expanded into resources

Pi starts with user and project settings, resolves package/local/CLI resources, loads extensions first, then skills, prompt templates, themes, context files, and system prompt fragments.

Config locations

User/global

Default global agent dir is ~/.pi/agent, overridden by PI_CODING_AGENT_DIR. It contains settings.json, auth.json, models.json, keybindings.json, extensions/, skills/, prompts/, themes/, sessions/.

Project

Project-local config lives under <cwd>/.pi/: settings, extensions, skills, prompts, themes, system prompt files, package installs, and git package checkouts.

CLI overrides

Flags can add or disable extensions, skills, prompt templates, themes, context files, and system prompt fragments. CLI resource paths are resolved before session cwd switches.

Settings merge

SettingsManager.create(cwd, agentDir) loads global and project settings, then deep-merges with project overriding global. Objects merge by key; arrays and primitives replace. Writes are scope-aware and lock-protected.

Gotcha: settings object merging is not a full semantic merge. Arrays such as resources replace; they do not concatenate across scopes.

Resource loader order

  1. Reload settings.
  2. Resolve package resources through DefaultPackageManager.
  3. Merge CLI/additional resource paths with settings/package/auto-discovered resources by canonical path.
  4. Load extensions with non-fatal diagnostics for failures and conflicts.
  5. Let extensions participate in resources_discover to add skills/prompts/themes.
  6. Load skills, prompt templates, and themes, deduping by name with first-wins precedence.
  7. Load context files from global and ancestor project AGENTS.md/CLAUDE.md-style files.
  8. Load replacement and append system prompt files.

Package/resource precedence

DefaultPackageManager sorts resources so closer and more explicit sources win:

project settingsproject autouser settingsuser autopackages

Package sources may be npm specs, git URLs/shorthands, or local paths. A package can declare resources in package.json#pi or by convention directories: extensions/, skills/, prompts/, themes/. Filters can include/exclude resources per package.

Loaded resource types

Extensions

TS/JS factories loaded by jiti. Can register tools, commands, shortcuts, flags, UI, providers, event hooks, and resources. Loaded before most resources so they can alter discovery.

Skills

Markdown SKILL.md files with frontmatter. Injected into the system prompt only when the read tool is active and model invocation is not disabled. May become slash commands when enabled.

Prompt templates

Markdown files invoked as slash commands. Support $1, $@, $ARGUMENTS, and slice substitutions for quoted arguments.

Themes

Built-ins plus JSON themes from user/project/packages. Theme schema validates colors and variables. User custom theme files can hot-reload.

Context files

Global and ancestor project files are loaded into system prompt construction so repository rules and local instructions travel with the session.

System prompts

Replacement SYSTEM.md and append APPEND_SYSTEM.md can come from project/global files or CLI text/path overrides.

Reload behavior

/reload refreshes settings-backed resources, keybindings, extensions, skills, prompt templates, themes, context files, and system prompt fragments. Because sessions and services are cwd-bound, resumed/forked sessions resolve resources relative to that session's cwd.

Keybindings

Keybindings are loaded from global keybindings.json only. The app merges TUI editor keybindings with app.* actions. Extensions can register shortcuts, but critical editor/app bindings are reserved.
